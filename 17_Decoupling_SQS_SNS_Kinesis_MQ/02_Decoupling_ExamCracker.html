SNS — Cheat Sheet

Pub/sub fan-out, push-based.

ConceptDetail
ModelPublish to a topic → SNS pushes a copy to every subscriber (one-to-many)
SubscribersSQS, Lambda, HTTP/S, email, SMS, mobile push, Kinesis Firehose
Fan-outSNS → multiple SQS = each consumer gets a durable buffer + retry + DLQ
FilteringPer-subscription filter policy → only relevant messages
SNS FIFOOrdered, deduplicated fan-out (with SQS FIFO)
Price$0.50/M publishes; SMS/email delivery extra
SNS vs EventBridge
SNS = simple, fast, high-throughput pub/sub. EventBridge = content-based routing + SaaS sources + schema registry + archive/replay + scheduling.

SQS vs Kinesis
SQS = consume & delete, competing consumers, no replay. Kinesis = retain & replay, many readers of the whole stream, ordered per shard. Real-time analytics/clickstream/IoT → Kinesis.
