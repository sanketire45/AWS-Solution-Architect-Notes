Query & Catalog · Part 1

The S3 Data Lake — The Foundation

Modern analytics starts by dumping all your raw data, cheaply, into one place: S3.

A data lake is a central repository where you store all your data — structured and unstructured, raw and processed — at any scale. On AWS, that's Amazon S3: cheap, durable, infinitely scalable.

You ingest data into S3 (via Kinesis Firehose, DataSync, direct upload), catalog its schema (Glue), then point various analytics engines at it — query it in place (Athena), process it (EMR), load it to a warehouse (Redshift), or visualise it (QuickSight). Decoupling storage (S3) from compute (the engines) is the core idea.

Ingest → S3 data lake → catalog schema with Glue → query/process with Athena/EMR/Redshift → visualise with QuickSight.
Query & Catalog · Part 2

Amazon Athena — Serverless SQL on S3

Athena runs standard SQL directly on data in S3 — no servers, no loading, no clusters. You pay only per query, based on data scanned. It's the go-to for ad-hoc analysis and "query my S3 data with least operational overhead."

🔑 The cost lever (tested)

Athena charges $5 per TB scanned. Partition your data (e.g. by date) and store it in columnar formats (Parquet/ORC) with compression so queries scan far less — turning a $5 query into cents. "Reduce Athena cost" → partitioning + Parquet.

💵 Athena = $5 / TB scanned, serverless. For occasional S3 analytics it's far cheaper than running a Redshift/EMR cluster 24/7.
Query & Catalog · Part 3

AWS Glue — Serverless ETL & the Data Catalog

  • Glue ETL — serverless Extract-Transform-Load jobs (Spark) to clean and reshape data, no servers to manage. "Serverless ETL, least ops" → Glue.
  • Glue Data Catalog — a central metadata/schema store used by Athena, Redshift Spectrum, EMR. Crawlers auto-discover schema from S3.
  • Glue DataBrew (visual data prep), Glue Studio (visual ETL).
Query & Catalog · Part 4

AWS Lake Formation — Build & Secure the Data Lake

Lake Formation sits on top of Glue/S3 to build a secure data lake quickly and apply fine-grained, centralized permissions (table-, column-, row-level) across analytics services. "Centrally govern who can access which columns/rows of the data lake" → Lake Formation.

🏢 Dublin Industry — Verified

Ryanair runs a company-wide S3 data lake fed by real-time Kinesis analytics

Ryanair (Dublin) ingests clickstream and operational events through Kinesis in real time into a company-wide S3 data lake, then analyses it downstream and feeds SageMaker for demand forecasting (famously predicting demand for "tea, ham and cheese paninis, and chocolate croissants" per route).

Coinbase stores PostgreSQL and DynamoDB snapshots in S3 feeding Databricks / Delta Lake — the same ingest → lake → process → analyse pipeline at crypto scale.

The canonical pipeline

Ingest (Kinesis Firehose) → store cheap (S3 lake) → catalog (Glue) → query ad-hoc (Athena) or warehouse (Redshift) → visualise (QuickSight). Decoupling storage from compute is what lets many teams analyse the same data independently.

Redshift · Part 1

Amazon Redshift — The Data Warehouse (OLAP)

Redshift is a columnar, massively-parallel data warehouse for running complex analytical queries (OLAP) over petabytes — the engine behind BI dashboards and reporting.

You load data into the cluster (from S3) and it's optimized for fast aggregations over huge tables. It's not for high-velocity transactional lookups (that's DynamoDB/RDS — OLTP).

⚠️ OLTP vs OLAP

"Millions of small transactions/lookups" = OLTP → DynamoDB/RDS. "Complex analytics/aggregations for BI" = OLAP → Redshift. Using Redshift as an app's operational DB (or RDS for petabyte analytics) is the classic mistake.

Redshift · Part 2

Spectrum & Serverless

  • Redshift Spectrum — query data directly in S3 from Redshift without loading it, joining lake + warehouse data.
  • Redshift Serverless — no cluster to manage; auto-scales and you pay for usage (RPU-hours).
  • Loads from S3 with COPY; integrates with Glue Catalog and QuickSight.
💵 Redshift from ~$0.25/hr (dc2.large) for a running cluster; Serverless ~$0.36/RPU-hr. A 24/7 cluster is pricey for occasional queries — that's where Athena wins.
Redshift · Part 3

Amazon EMR OpenSearch Triggers
EMR · Part 1

Amazon EMR — Managed Big-Data Frameworks

EMR (Elastic MapReduce) is managed Hadoop, Spark, Hive, Presto, HBase — for processing vast datasets (petabytes) with the big-data ecosystem.

Use it for heavy custom data processing, ETL at massive scale, ML data prep, and Spark/Hadoop jobs that don't fit Athena's simple SQL model. EMR provisions a cluster (EC2) — use Spot for task nodes to cut cost dramatically.

🔑 EMR vs Athena vs Glue

Athena = serverless SQL on S3 (simplest). Glue = serverless Spark ETL + catalog. EMR = full Hadoop/Spark cluster for the heaviest custom big-data processing (most control, you manage the cluster). "Run Spark/Hadoop on petabytes" → EMR.

💵 EMR = EC2 cost + a small per-instance EMR surcharge; Spot task nodes can cut the bill ~70–90% for fault-tolerant jobs.
OpenSearch · Part 1

Amazon OpenSearch Service — Search & Log Analytics

  • Managed OpenSearch/Elasticsearch for full-text search and log & observability analytics with dashboards (Kibana/OpenSearch Dashboards).
  • Canonical least-ops logging pipeline: CloudWatch Logs / Kinesis Firehose → OpenSearch → Dashboards.
  • "Full-text search" or "analyse and visualise logs in near-real-time" → OpenSearch.

Kinesis Recap Amazon MSK (Kafka) QuickSight (BI) Triggers
Streaming · Part 1

Kinesis in the Analytics Pipeline (Recap)

  • Kinesis Data Streams — real-time ingest, ordered, replayable (covered in Decoupling).
  • Kinesis Data Firehose — near-real-time delivery to S3 / Redshift / OpenSearch, least ops. The standard "stream → data lake" path.
  • Managed Service for Apache Flink (Kinesis Data Analytics) — real-time SQL/aggregations on streams.
Streaming · Part 2

Amazon MSK — Managed Apache Kafka

MSK (Managed Streaming for Apache Kafka) is fully-managed Apache Kafka — for teams already using Kafka or needing its ecosystem at scale. Kinesis vs MSK: Kinesis is AWS-native and simplest; MSK is the answer when "Apache Kafka" is explicitly required or you're migrating Kafka workloads. (Coinbase, Flutter, and Ryanair all run MSK at scale.)

🔑 Kinesis vs MSK

"Real-time streaming, AWS-native, least ops" → Kinesis. "Apache Kafka specifically / migrate existing Kafka" → MSK (MSK Serverless removes broker management).

💵 MSK bills per broker-hour (+ storage), always-on — like Amazon MQ. MSK Serverless charges per partition + throughput.
Streaming · Part 3

Amazon QuickSight — Serverless BI

  • Serverless business-intelligence dashboards over Redshift, Athena, RDS, S3, and more.
  • SPICE in-memory engine for fast interactive analysis; ML Insights for anomaly detection & forecasting.
  • Per-user (author) or per-session (reader) pricing — the visualisation layer of the analytics stack.
💵 QuickSight ≈ $24/author/month (or $18 annual); readers ~$0.30/session (capped $5/mo). Pay-per-session reader pricing suits dashboards viewed occasionally by many people.
