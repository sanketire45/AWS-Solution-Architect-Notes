KMS · Part 1

Why KMS — Managed Encryption Keys

AWS KMS (Key Management Service) creates and controls the encryption keys used to protect your data, integrated into almost every AWS service (S3 SSE-KMS, EBS, RDS, Secrets Manager…).

You never see the raw key material — KMS uses it on your behalf and logs every use in CloudTrail. Access is governed by key policies (+ IAM). It uses envelope encryption: KMS encrypts a data key, which encrypts your data, so you can encrypt large objects efficiently.

KMS · Part 2

Key Types, Policies & Rotation

  • AWS-owned (invisible, free), AWS-managed (aws/service, free, auto-rotate), and Customer-managed (CMK) — full control of policy, rotation, audit.
  • Automatic rotation — customer-managed keys can auto-rotate yearly; "encrypt with a key we rotate annually, with full audit/control" → customer-managed CMK + rotation (the SSE-S3 distractor lacks control).
  • Key policy defines who can use/administer the key (required for cross-account key use).
  • Multi-Region keys for cross-region replicated encrypted data; symmetric (default) vs asymmetric keys.
💵 AWS-managed keys are free; a customer-managed CMK is $1/month + $0.03 per 10,000 requests. Cheap, and the audit/control is usually worth it for sensitive data.
KMS · Part 3

TenancyMulti-tenant, AWS-managedSingle-tenant dedicated hardware

🔑 Trigger

"Managed keys, integrate with AWS services" → KMS. "Dedicated, single-tenant hardware security module / we must fully control key material for compliance" → CloudHSM.

🏢 Dublin Industry — Verified / Inferred

How fintechs lock down cardholder data — the Stripe & Ryanair security stack

Stripe isolates its Card Data Vault in a dedicated, hardened AWS environment with AES-256 encryption, CloudHSM + KMS for key control, Secrets Manager for credentials, and GuardDuty for threat detection — PCI DSS Level 1, tokenization, and tight blast-radius isolation.

Ryanair fronts customer endpoints with the standard secure edge: CloudFront + WAF + Shield Advanced, KMS for PII at rest, Secrets Manager for DB creds, and GuardDuty/Security Hub for posture.

The fintech security blueprint

Encrypt everything (KMS, or CloudHSM where you must own key material) · isolate sensitive data in its own account/VPC · rotate secrets automatically (Secrets Manager) · block at the edge (WAF + Shield) · detect continuously (GuardDuty + Macie + Security Hub). This is the architecture the Security domain (30%) tests.

Secrets · Part 1

Secrets Manager vs SSM Parameter Store

Both store secrets securely. The difference is rotation (and cost).

Secrets ManagerParameter Store
StoresSecrets (DB creds, API keys)Config + secrets (SecureString via KMS)
Automatic rotation✅ Built-in (native for RDS)❌ No native rotation
Cost$0.40/secret/monthFree (standard tier)
Use whenNeed automatic rotation of secretsPlain config / secure strings, cost-sensitive
🔑 The deciding factor

Need automatic credential rotation (especially for RDS) → Secrets Manager. Just need to store config/secrets cheaply with no rotation → Parameter Store (free). Either way, never hard-code credentials in code/AMIs.

Secrets · Part 2

AWS Certificate Manager (ACM)

  • Free public TLS/SSL certificates that auto-renew — no more expired-cert outages.
  • Used on ALB, NLB (TLS), CloudFront, API Gateway — never installed directly on EC2.
  • CloudFront certs must be in us-east-1 (the recurring trap); ALB certs in the ALB's region.
  • Imported certs don't auto-renew. ACM Private CA for internal/private certificate hierarchies.

WAF · Part 1

AWS WAF — Web Application Firewall (Layer 7)

  • Filters HTTP/HTTPS traffic with a Web ACL: block SQL injection, XSS, bad bots, apply rate-based rules, geo & IP match, and managed rule groups (incl. Bot Control).
  • Attaches to: CloudFront, ALB, API Gateway (REST), AppSync, Cognito User Pools, App Runner — NOT NLB (a classic trap).
  • "Block SQLi/XSS, rate-limit, filter bots at the application layer" → WAF.
⚠️ Trap

WAF cannot attach to a Network Load Balancer (NLB) — it's Layer 7. For NLB/L4 protection you rely on Shield + security groups/Network Firewall.

Shield · Part 2

AWS Shield — DDoS Protection

Shield StandardShield Advanced
CostFree (automatic)$3,000/month (1-yr commit)
ProtectionCommon network/transport (L3/L4) DDoSAdvanced DDoS + larger attacks
Extras—DDoS cost protection (refunds scaling charges), SRT (response team), advanced reporting, free WAF on protected resources
🔑 Trigger

"DDoS cost protection + 24/7 response team for a critical app" → Shield Advanced. Baseline DDoS is already covered free by Shield Standard on CloudFront/Route 53/Global Accelerator.

WAF & Shield · Part 3

The Standard Global-App Protection Stack

The exam's go-to secure edge architecture: CloudFront + AWS WAF + Shield (+ Shield Advanced for critical apps) in front of your origin. WAF blocks malicious requests at the edge; Shield absorbs DDoS; CloudFront caches and provides the edge. For multi-account WAF/Shield/SG rule management, use AWS Firewall Manager.

Detection · Part 1

The Big Three — GuardDuty · Inspector · Macie

Three "find the problem" services the exam loves to confuse.

ServiceFindsHow / trigger
GuardDutyThreats & malicious activityML on logs (VPC Flow, CloudTrail, DNS, EKS, RDS login, Lambda) — agentless. "Detect threats/anomalies with ML"
InspectorVulnerabilities (CVEs)Scans EC2, ECR container images, Lambda for software vulns & misconfig. "Find vulnerabilities"
MacieSensitive data (PII)ML discovers/classifies PII in S3. "Find PII / sensitive data in S3"
🔑 Memory hook

GuardDuty = threats (logs). Inspector = vulnerabilities (CVEs). Macie = sensitive data (S3 PII).

Detection · Part 2

Security Hub & Detective

  • Security Hub — aggregates & prioritizes findings from GuardDuty, Inspector, Macie, etc. into one dashboard with compliance standards (CIS, PCI). "Central security posture / aggregate findings" → Security Hub.
  • Detective — investigate & root-cause a finding (visualizes relationships across logs). "Deep-dive/investigate the root cause of a finding" → Detective.
Detection · Part 3

Network Firewall & Firewall Manager

  • AWS Network Firewall — managed, stateful network/IDS-IPS firewall for your VPC (deep packet inspection, domain filtering).
  • AWS Firewall Manager — centrally manage WAF rules, Shield Advanced, security groups, and Network Firewall policies across all accounts in an Organization.
