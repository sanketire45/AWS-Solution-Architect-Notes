Exam Triggers & Traps

If the question says X → the answer is Y. Traps in red.

Storage Classes

"Milliseconds, archive, quarterly access"→Glacier Instant Retrieval
"Minutes to hours acceptable, archive"→Glacier Flexible Retrieval
"12–48 hours acceptable, compliance archive, cheapest"→Glacier Deep Archive
"Unknown access pattern, auto optimize"→Intelligent-Tiering
"Infrequent, but can lose data if AZ fails"→One Zone-IA (acceptable for re-creatable data)
⚠ TRAP: "Archive tiers in Intelligent-Tiering activate automatically"→FALSE — Archive Access and Deep Archive tiers must be manually enabled
⚠ TRAP: "Delete Standard-IA object after 10 days, pay 10 days"→FALSE — minimum 30-day billing still applies

Versioning, Object Lock & Replication

"Protect against accidental deletion"→Enable Versioning
"Cannot be deleted by anyone, 7-year compliance"→Object Lock — Compliance Mode
"WORM but privileged users can override"→Object Lock — Governance Mode
"Preserve data for litigation, no fixed date"→Object Lock — Legal Hold
"WORM on standalone Glacier vault"→Glacier Vault Lock Policy
"DR copy in another region" / "comply with data residency in second region"→Cross-Region Replication (CRR)
"Aggregate logs from multiple buckets to one"→Same-Region Replication (SRR)
"Guaranteed 15-minute replication RPO SLA"→S3 Replication Time Control (RTC)
⚠ TRAP: "Enable CRR → existing objects replicate"→FALSE — only new objects. Use S3 Batch Replication for existing.
⚠ TRAP: "A→B replication + B→C replication means A→C"→FALSE — replication is not transitive

Performance

"Upload large file from distant location, slow/fails"→Multipart Upload + S3 Transfer Acceleration
"S3 GET requests hitting throughput limits"→Distribute objects across multiple prefixes
"Need only certain rows from a large CSV in S3"→S3 Select (SQL-like query on S3)
"Abandoned multipart uploads costing money"→Lifecycle rule: AbortIncompleteMultipartUpload
"KMS throttling errors with SSE-KMS at scale"→Enable S3 Bucket Keys

Encryption

"Simplest, AWS manages keys, default"→SSE-S3 (default since Jan 2023)
"Audit who used which key, key rotation control"→SSE-KMS (CloudTrail logs every KMS call)
"You provide encryption keys, AWS never stores them"→SSE-C (HTTPS is mandatory)
"Data must be encrypted BEFORE it reaches AWS"→Client-Side Encryption
"Force all PUT operations to be encrypted"→Bucket Policy — Deny PUT without encryption header
⚠ TRAP: "SSE-C = client-side encryption"→FALSE — SSE-C still sends plaintext to AWS. Client-side = you encrypt first, AWS never sees plaintext.

Access Control, Security & Networking

"Ensure no bucket in account is ever public"→Block Public Access at account level
"Allow user to download private file for 1 hour"→Presigned URL
"JS on one domain loading files from another S3 domain"→CORS configuration on S3
"Private subnet EC2 accessing S3, no internet, free"→VPC Gateway Endpoint for S3 (free, route table only)
"Connect to S3 via PrivateLink / other AWS services"→VPC Interface Endpoint (paid, ~$0.01/hr/AZ)
"External downloaders pay their own transfer costs"→Requester Pays (authenticated users only — no anonymous)
⚠ TRAP: "Presigned URL always valid until its expiry time"→FALSE — URL expires when the creator's IAM credentials expire, which may be sooner
⚠ TRAP: "Explicit ALLOW in bucket policy overrides DENY in IAM"→FALSE — Explicit DENY always wins everywhere

Logging, Monitoring & Consistency

"Who deleted the S3 bucket / changed policy?"→CloudTrail (management events)
"Which IP address downloaded a specific object?"→S3 Server Access Logging
"Guaranteed capture of every object-level access event"→CloudTrail Data Events (paid, not Server Access Logs)
"S3 usage trends across all accounts in Organisation"→S3 Storage Lens
"What consistency model does S3 use?"→Strong consistency (since December 2020, for all operations)
⚠ TRAP: "Log S3 access to the same bucket"→NEVER do this — creates infinite logging loop. Use a separate destination bucket.
