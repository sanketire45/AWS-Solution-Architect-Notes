Storage Gateway — Cheat Sheet

Hybrid: on-prem interface, cloud backend.

TypeInterfaceBackend / use
File GatewayNFS / SMBFiles → S3 objects (+ local cache); FSx File GW → FSx Windows
Volume Gateway — CachediSCSIPrimary data in S3, hot cached locally
Volume Gateway — StorediSCSIPrimary data on-prem, async EBS-snapshot backup to S3
Tape GatewayiSCSI VTLVirtual tapes → S3/Glacier; keep backup app
Gateway vs DataSync
Storage Gateway = ongoing hybrid cached access. DataSync = one-time/scheduled bulk migration. Don't swap them.

ServiceWhatLands in
