The Monitoring & Audit Cheat Sheet

Match the clue to the service. The three-way distinction is the whole section.

Clue in the questionService
Metrics, alarms, dashboards, performanceCloudWatch
Collect EC2 memory / disk-space metricsCloudWatch Agent (not default)
Alert / auto-scale on a thresholdCloudWatch Alarm → SNS / ASG
Auto-recover a failed EC2 instanceCloudWatch Alarm (EC2 recover action)
React to an AWS event / scheduled cronEventBridge
Trace requests across microservicesX-Ray
Who made an API call / who deleted X (audit)CloudTrail
Audit access to individual S3 objectsCloudTrail data events (not default)
Central audit log across all org accountsCloudTrail Organization Trail
Prove logs were not tampered withCloudTrail log file validation
Is this resource compliant? / config historyAWS Config
Auto-fix non-compliant resourcesConfig rule + SSM remediation
