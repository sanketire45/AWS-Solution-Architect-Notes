Vision · Part 1

AWS AI Services — Pre-Trained, API-Driven

Most AWS "ML" is not building models — it's calling a ready-made AI API.

You don't need a data-science team to add AI. AWS provides pre-trained, fully-managed AI services you call like any API: send an image, get back labels; send audio, get back text. No training, no servers.

For the exam, the skill is mapping a problem to the right service. Build-your-own (SageMaker) is for when no pre-built service fits and you have your own data/model.

Vision · Part 2

Amazon Rekognition — Image & Video Analysis

  • Detect objects, scenes, faces, text in images and video.
  • Facial analysis & comparison (identity verification), celebrity recognition.
  • Content moderation — flag inappropriate/unsafe images & video (a very common exam use case).
  • "Analyse images/video, detect faces/objects, moderate content" → Rekognition.
Vision · Part 3

Amazon Textract — Extract Text & Data from Documents

Textract goes beyond plain OCR: it extracts text, tables, forms (key-value pairs) from scanned documents, PDFs, invoices, and IDs — preserving structure. "Extract data/tables/forms from scanned documents" → Textract.

🔑 Rekognition vs Textract

Rekognition = analyse images/video (faces, objects, moderation). Textract = extract structured text/forms/tables from documents. "Read a scanned invoice's fields" → Textract, not Rekognition.

🔑 Chain them

A common pattern: Transcribe (call audio → text) → Comprehend (sentiment/entities) → Translate (other languages) → Polly (back to speech). Recognise the building blocks.

Language · Part 2

Amazon Comprehend — Natural Language Processing

  • NLP on text: sentiment, entities, key phrases, language detection, topic modeling.
  • Detect & redact PII in text; Comprehend Medical for clinical text.
  • "Analyse sentiment / extract entities / find PII in text" → Comprehend.
Language · Part 3

Amazon Lex — Conversational Chatbots

Lex builds conversational interfaces (chatbots / voice bots) using the same technology as Alexa — automatic speech recognition + natural-language understanding to capture intent. Pairs with Amazon Connect (contact center) for IVR. "Build a chatbot / voice assistant / IVR" → Lex.

SageMaker · Part 1

Amazon SageMaker — Build Your Own ML

When no pre-built AI service fits and you have your own data and model, SageMaker is the full workbench.

SageMaker is the end-to-end platform to build, train, tune, and deploy your own machine-learning models — managing the heavy lifting of notebooks, training infrastructure, and hosting.

For the exam: when a scenario needs a custom model trained on the company's own data (not a pre-trained API like Rekognition/Comprehend), the answer is SageMaker.

SageMaker · Part 2

The Pieces You Should Recognise

  • SageMaker Studio — the IDE; notebooks for exploration.
  • Ground Truth — data labeling (human + ML-assisted).
  • Training jobs + automatic model tuning; built-in algorithms & JumpStart (pre-built models).
  • Endpoints — deploy a model for real-time inference (or batch transform).
  • Autopilot / Canvas — auto/no-code ML; Feature Store, Pipelines, Model Monitor.
🔑 Pre-built service vs SageMaker

"Standard task (faces, sentiment, transcription)" → use the pre-trained AI service (least effort). "Custom model on our own dataset / unique problem" → SageMaker. Don't reach for SageMaker when Rekognition/Comprehend already solves it.

Specialized · Part 2

Fraud Detector & Bedrock

  • Amazon Fraud Detector — detect online fraud (fake accounts, payment fraud) with ML, no ML expertise needed.
  • Amazon Bedrock — the newer generative-AI service: access foundation models (Anthropic Claude, etc.) via API to build GenAI apps. "Generative AI / foundation models / build a chatbot with an LLM / RAG" → Bedrock.
  • Comprehend Medical / Transcribe Medical — healthcare-specific variants.
